# UI/UXレビュー報告: ゲスト公開コンテンツ拡張

## 1. 概要

- 対象PR / 作業: Example / Reader / Quiz のゲスト公開対応
- 変更した画面・コンポーネント: Examples、Reader ArticleListPanel、QuizPage
- 判定: Pass
- P0件数: 0
- P1件数: 0
- P2件数: 1

## 2. ユーザー価値

- 対象ユーザー: 公開教材をログインなしで閲覧する学習者、公開範囲を管理するログイン済みユーザー
- 利用文脈: 公開済み WordPack だけでなく、関連する例文、Reader 記事、Quiz も共有したい場面
- ユーザー目的: ゲストは安全に公開教材を読む。ログイン済みユーザーはReader記事とQuizを公開/非公開へ切り替える。
- 支援するタスク: 公開コンテンツの閲覧、公開範囲の確認、Reader / Quiz 単位の公開設定変更
- このUIが助ける理解・判断・行動: 「公開中」「非公開」表示と「公開にする」「非公開にする」ボタンで、現在の公開状態と次の操作が分かる。
- このUIがなければ困る点: WordPack以外の教材をゲストへ共有できず、公開範囲も画面上で判断できない。
- 削るべき情報・操作: ゲストには公開切替ボタンを表示しない。管理できない操作を見せず、閲覧範囲だけに集中させる。
- 検証仮説・成功指標: ゲストでは公開済みコンテンツだけが表示され、ログイン済みユーザーはReader / Quizの公開状態を1操作で切り替えられる。

## 3. 初見理解

- 何の画面か分かるか: 既存の Examples / Reader / Quiz の見出しを維持。
- 今どこか分かるか: 既存ナビゲーションを維持。
- 何ができるか分かるか: ログイン済みユーザーには公開状態ラベルと公開切替ボタンを表示。
- 最初の有意味な行動: ゲストは一覧から公開コンテンツを開く。ログイン済みユーザーは対象カードの公開ボタンを押す。
- 操作結果を予測できるか: ボタン文言が次の状態を示し、成功時はステータスメッセージを表示。
- 失敗時に戻れるか: 公開切替は楽観更新後にAPI失敗時ロールバックし、エラーメッセージを表示。

## 5. アクセシビリティ確認

- キーボード: Reader / Quiz の公開切替は通常の button。ゲスト時は管理ボタンを非表示にして到達不能な操作を残さない。
- フォーカス: Reader 公開ボタンに `:focus-visible` を追加。Quiz は既存の `quiz-secondary-button` focus-visible に含まれる。
- 名前・ラベル: ボタンは「公開にする」「非公開にする」の可視テキストを持つ。
- 見出し・構造: 既存の画面見出しとregion構造を維持。
- コントラスト: Readerの状態pillは淡色背景に濃色文字、Quizは既存ダークUI上の高コントラスト色を使用。
- ターゲットサイズ: Readerボタンは `min-height: 2.3rem`、Quizボタンは既存の `min-height: 2.3rem`。
- エラー・ステータス: 成功はstatus、失敗はalertで表示。
- 自動検査: Vitest / typecheck / Playwright smoke を実行。
- 手動確認: コード上でゲスト時の操作非表示、空状態文言、ロールバックを確認。

## 6. 視覚階層

- 主操作: 公開切替は各カード内の公開状態ラベルの隣に配置。
- 情報優先度: タイトル、状態、更新日時の順を維持。
- グルーピング: 公開状態と切替は `.al-public-row` / `.quiz-public-row` で同じ意味のまとまりにした。
- 余白・密度: 既存カード密度を大きく変えず、公開行は折り返し可能。
- 読みやすさ: ボタンは短い動詞句、状態は2-3文字のラベル。
- 狭幅・文字拡大: flex-wrap によりボタンとpillが折り返せる。

## 7. コピー

- 用語: WordPack、Example、Reader、Quiz、ゲスト公開を既存文書に合わせた。
- ボタン・リンク: 「公開にする」「非公開にする」で次の操作を明示。
- エラー文: API失敗時は対象コンテンツ名を含める。
- 空状態: ゲストとログイン済みで理由と次の理解が分かれる文言にした。
- disabled: 新規公開切替はゲスト時にdisabled表示せず、非表示にした。
- トーン: ユーザーを責めず、公開されていない事実だけを説明。

## 8. 熟練者効率

- 主要反復タスク: Reader / Quiz 一覧から1クリックで公開状態を切り替える。
- 手数: 詳細画面へ入らずカード上で完結。
- 再入力・再選択: なし。
- 近道: 一括公開は今回追加しない。既存WordPackの公開操作と同じ一覧内操作に寄せた。
- 初心者向け説明の影響: 空状態以外に恒常的な説明文を増やしていない。
- 判定: Pass

## 9. 満足感・信頼感

- 待機中: 楽観更新により即時反応する。
- 成功時: 公開/非公開の完了メッセージを表示。
- 失敗時: ロールバックして失敗理由を表示。
- 危険操作: 公開範囲を広げる操作のため、現在状態と次操作を隣接表示。削除のような破壊的操作ではないため確認ダイアログは追加しない。
- データ・権限・個人情報: ゲストは公開済みデータのみ閲覧可能。Reader記事内の関連WordPackも公開済みだけ返す。
- トーン: 不安を煽らず、公開状態を明確にする。
- 判定: Pass

## 10. 反証レビュー

- 実装を落とす観点で見つけた問題: ゲスト時に公開切替ボタンをdisabled表示すると管理できる範囲を誤認させるため、非表示に修正済み。
- P0候補: 非公開Reader記事や非公開Quizがゲスト詳細で見える問題はテストで404を確認済み。
- 証跡不足: 専用スクリーンショットは未取得。既存Playwright smokeとDOMテストで主要操作を確認。
- 残リスク: Exampleのゲストフィルタは親WordPackを参照してアプリ側で絞るため、大量データ時の読み取り量は増える可能性がある。
