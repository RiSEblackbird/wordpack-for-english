# Visual Hierarchy Checklist

- [ ] Screen title or purpose is clear.
- [ ] Current object/scope/filter/mode is clear.
- [ ] One primary action dominates the decision area.
- [ ] Secondary actions are visually secondary.
- [ ] Destructive actions are visually distinct and risk-appropriate.
- [ ] Related items are grouped.
- [ ] Alignment supports scanning.
- [ ] Whitespace separates regions.
- [ ] Metadata does not compete with task-critical content.
- [ ] Counts/badges/status indicators explain their scope.
- [ ] Long labels do not break layout.
- [ ] Narrow viewport remains usable.
- [ ] Text is readable at expected sizes.
- [ ] The layout survives empty, many, and error states.
