# Cognitive Walkthrough Checklist

For each changed screen/component:

- [ ] The user goal is visible or inferable from visible UI.
- [ ] The first meaningful action is visible.
- [ ] The action label predicts the result.
- [ ] The user does not need documentation.
- [ ] The user does not need to remember hidden prior information.
- [ ] Current location/scope is visible.
- [ ] Feedback after action is visible.
- [ ] Error recovery is visible.
- [ ] Empty state gives a next step.
- [ ] Disabled state explains what enables it.
- [ ] Advanced options do not block the beginner path.
- [ ] Copy uses user terminology.
- [ ] The UI remains understandable after interruption.
