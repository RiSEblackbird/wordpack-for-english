# P0 / P1 / P2 チェックリスト

## P0: 完了不可

- [ ] ユーザー価値、対象ユーザー、支援する目的を説明できない。
- [ ] 初見で画面目的が分からない。
- [ ] 現在地、対象範囲、選択中、適用範囲が分からない。
- [ ] 最初の有意味な行動が分からない。
- [ ] 主操作がアイコン単独、または視覚的に埋もれている。
- [ ] 主要状態が混ざっている。
- [ ] エラーに原因、影響、回復手段がない。
- [ ] disabledの理由、有効化条件が分からない。
- [ ] キーボード操作、フォーカス、ラベル、名前、コントラスト、ターゲットサイズ、構造に重大問題がある。
- [ ] 色だけで意味を伝えている。
- [ ] 主要反復タスクが、不要な説明、確認、再入力、再選択で妨害される。
- [ ] 危険操作、削除、送信、公開、課金、権限、個人情報に関わるUIが信頼できない。
- [ ] 成功、失敗、待機の結果が分からない。
- [ ] ユーザーを責める、または不要な不安を煽る。
- [ ] state matrix、初見シミュレーション、反証レビュー、証跡がない。
- [ ] 未実行検証を成功扱いしている。

## P1: 原則同じ変更内で修正

- [ ] 検索、タブ、フィルタ、件数のスコープが曖昧。
- [ ] 重要度と視覚的強調が少しズレている。
- [ ] 用語やラベルに揺れがある。
- [ ] 空状態、成功状態、エラー状態の次アクションが弱い。
- [ ] 反復作業の手数が多い。
- [ ] 初心者向け説明が少し冗長。
- [ ] 待機、成功、失敗のフィードバックがやや弱い。

## P2: Issue化可

- [ ] 余白、文言、順序の微改善。
- [ ] 追加の近道、一括操作、ショートカット。
- [ ] より良い補助説明、空状態、成功状態。
- [ ] 追加調査や計測が必要な改善仮説。
