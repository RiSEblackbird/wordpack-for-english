# 満足感・信頼感チェックリスト

- [ ] 待機中であることが分かる。
- [ ] 待機中に既存データが消えたように見えない。
- [ ] 成功時に何が完了したか分かる。
- [ ] 失敗時に原因、影響、回復手段が分かる。
- [ ] 入力内容や作業内容が保持されるか分かる。
- [ ] 危険操作の対象、件数、影響、取り消し可否が分かる。
- [ ] 削除、送信、公開、課金、権限、個人情報の扱いが明確。
- [ ] UIコピーがユーザーを責めていない。
- [ ] 警告が過剰に怖くない。
- [ ] 安心表現が実態より強すぎない。
- [ ] 成功か失敗か曖昧な通知がない。
- [ ] 操作後に次の見通しがある。
